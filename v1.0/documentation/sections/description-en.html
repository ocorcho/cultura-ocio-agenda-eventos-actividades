La siguiente figura muestra el diagrama de clases de la ontología, donde se puede observar que el elemento central es la clase esagenda:Evento, definida en este vocabulario

En la figura también se muestran todas las propiedades definidas en este vocabulario, para representar el lugar o equipamiento de la ciudad donde se celebrará el evento, el lugar de inscripción, las características de accesibilidad y medios de transporte para llegar, los tipos de público y de evento, y las fechas y horas de inicio y fin de las actividades. También se muestran las propiedades que se pueden utilizar de otros vocabularios importados, como la Event ontology y schema.org/Event

También se pueden utilizar, y se recomienda utilizar, propiedades generales de Dublin Core (por ejemplo, dct:title y dct:identifier) y otras propiedades generales como rdfs:label, rdfs:comment, owl:sameAs, etc.

3.1 Ejemplos de uso back to ToC

A continuación mostramos algunos ejemplos del uso de algunas de estas clases y propiedades, con ejemplos procedentes del portal de datos abiertos de Alcobendas y Zaragoza. Utilizaremos en todos estos ejemplos el formato Turtle

Comenazamos con el portal de Alcobendas. Aquí puede verse como se representa un evento de la agenda con sus propiedades asociadas. 

        <http://datos.alcobendas.org/recurso/agenda/actividad/8865642fd77b9f7b62bec59e56aec0dd> ;
        a			<http://vocab.linkeddata.es/datosabiertos/def/cultura-ocio/agenda#Evento> ;
	schema:description      "Datos de interés Descripción Puede consultar el folleto con todos los espectáculos programados para el Primer período de la temporada. Puede informarse sobre abonos y horarios." ;
	schema:name		"(2º Período: Febrero-Mayo 2018) Programación cultural Revista El Público" ;
	esagenda:fechaFin	"2018-05-26" ;
	esagenda:fechaInicio	"2018-02-03" ;
	esagenda:tipoEvento	<http://datos.localidata.com/api/alcobendas/kos/commons/tema/culturayocio/teatromusicaydanza> ;
        rdfs:label              "(2º Período: Febrero-Mayo 2018) Programación cultural Revista El Público" .
